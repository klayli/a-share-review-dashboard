# 交易日早报自动生成 Spec

## Why

`MORNING_REPORTS` 是完全硬编码的静态对象，每天早上需要手动编写早报数据。用户需要在每个交易日早上 9:00 前自动生成一份包含隔夜外盘、A股前瞻、要闻精选等内容的金融市场早报，无需人工干预。

## What Changes

- **新增**：Cloudflare Worker 定时任务（Cron Trigger），交易日 8:30 执行
- **新增**：早报自动生成逻辑 — 从东方财富/新浪 API 拉取隔夜美股、A股前日收盘、商品外汇等数据，生成结构化早报 JSON
- **新增**：Worker 将生成的早报写入 Cloudflare KV，前端从 KV 读取
- **修改**：前端 `getMorningReport()` 改为优先从 KV API 获取当日早报，无 KV 数据时 fallback 到 `MORNING_REPORTS` 静态数据

## Impact

- Affected code: `a-share-review-dashboard.html` — `getMorningReport()` 函数
- 新增：Cloudflare Worker 代码（`worker/` 目录）
- 新增：Cloudflare KV namespace 用于存储早报数据
- 不影响现有 `MORNING_REPORTS` 静态数据（作为 fallback）

## 方案设计

### 架构

```
Cloudflare Worker (Cron: 每个交易日 8:30 CST)
  ├── 1. 判断是否为交易日（排除周末+节假日）
  ├── 2. 调用东方财富 API 获取数据
  │     ├── 隔夜美股三大指数（道琼斯/标普/纳斯达克）
  │     ├── A股前日收盘数据（上证/深证/创业板）
  │     ├── 恒生指数
  │     ├── 商品（黄金/原油/铜）
  │     └── 外汇（美元指数/离岸人民币）
  ├── 3. 调用 AI 生成早报文字内容
  │     ├── 核心关注（今日焦点）
  │     ├── 要闻精选（5-8条）
  │     └── 风险提示
  └── 4. 写入 KV: MORNING_REPORT_{YYYY-MM-DD}

前端页面
  ├── getMorningReport(date)
  │     ├── 优先 fetch /api/morning/{date}.json (KV)
  │     └── fallback → MORNING_REPORTS[date] (静态)
  └── 展示早报
```

## ADDED Requirements

### Requirement: 交易日自动早报生成

The system SHALL automatically generate a morning report JSON on each trading day at 8:30 AM (CST), by fetching real market data from Eastmoney API and using AI to generate textual content.

#### Scenario: 交易日早上自动生成
- **WHEN** 今天是交易日且时间为 8:30 AM CST
- **THEN** Cloudflare Worker Cron 触发，拉取隔夜美股、A股数据、商品外汇
- **AND** 调用 AI 生成核心关注、要闻精选、风险提示
- **AND** 将完整早报 JSON 写入 KV `MORNING_REPORT_{date}`

#### Scenario: 非交易日跳过
- **WHEN** 今天是周六/周日或法定节假日
- **THEN** Worker 不执行，不生成早报

### Requirement: 前端动态读取早报

The system SHALL fetch morning report from KV API first, falling back to static `MORNING_REPORTS` data.

#### Scenario: KV 有当日早报
- **WHEN** 用户访问首页，当日为 2026-07-08
- **THEN** `getMorningReport('2026-07-08')` 从 `/api/morning/2026-07-08.json` 获取 KV 数据
- **AND** 渲染完整早报卡片

#### Scenario: KV 无数据，fallback 到静态
- **WHEN** 用户访问首页，KV 无当日数据
- **THEN** fallback 到 `MORNING_REPORTS['2026-07-08']` 静态数据
- **AND** 若静态也没有，显示"当日早报待更新"

## MODIFIED Requirements

### Requirement: getMorningReport 函数

**Before**: 仅从 `MORNING_REPORTS` 静态对象查找
**After**: 优先 fetch KV API，无数据时 fallback 到静态数据
