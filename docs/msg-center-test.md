# A股短线复盘看板 - 消息中心功能全量测试用例

---

## 1. 测试策略与范围

### 1.1 测试目标
验证"消息中心"功能在各种场景下的正确性、稳定性与用户体验，确保消息入口可见性、列表展示、详情展示、已读状态持久化及交互关闭等核心链路无缺陷。

### 1.2 测试范围
| 范围项 | 说明 |
|--------|------|
| 功能测试 | 入口角标、消息列表、消息详情、已读状态管理、交互与关闭 |
| 兼容性测试 | 浏览器兼容性、移动端适配、主题切换（暗色/亮色） |
| 性能测试 | 消息渲染耗时、大数据量列表滚动、面板打开/关闭动画 |
| 异常测试 | 无消息场景、localStorage异常、数据缺失、边界值 |
| 回归测试 | 消息中心与主页面其他模块的相互影响 |

### 1.3 测试类型
- 黑盒功能测试（为主）
- 灰盒测试（结合代码逻辑验证localStorage键值、DOM操作）
- 手工测试 + 浏览器DevTools辅助验证
