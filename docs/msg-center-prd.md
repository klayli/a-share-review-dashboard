## 1. 需求背景与目标

### 1.1 背景

A股短线复盘看板是一个纯前端单页应用，面向短线投资者提供持仓总览、大盘情绪、趋势图表、交易记录等核心数据。当前用户查看历史每日收益变动时，需要在多个页面或数据卡片间手动翻找，无法快速定位某一天的完整复盘摘要。

看板的数据来源为静态 JSON 文件（portfolio.json）与内置 JS 变量，没有服务端推送能力。每日的收益摘要数据随静态资源一并发布，用户打开页面即可获取全量历史数据。

### 1.2 目标

消息中心功能的目标是让用户在任意页面都能通过统一的入口，快速查看历史每日收益摘要，区分已读与未读，并支持一键标记全部已读。用户无需跳转路由或离开当前上下文，即可完成历史复盘信息的浏览。

具体目标如下：

- **减少操作路径**：将"查找某日收益"的点击次数从 3 次以上降低到 2 次（打开面板 + 点击消息）。
- **强化未读感知**：通过数字角标提醒用户有新的每日复盘数据尚未查看。
- **支持闭环浏览**：列表概览与详情阅读在同一面板内完成，提供返回机制。

### 1.3 问题定义

若不做消息中心，用户面临的实际问题是：

- 静态 JSON 中已包含历史每日数据，但缺乏一个按时间聚合的浏览入口。
- 用户难以判断"自上次打开以来，有哪些新的交易日数据需要关注"。
- 大盘情绪页和持仓详情页各自分散展示当日信息，没有统一的摘要视图。

## 2. 用户故事

### 2.1 核心用户画像

| 维度 | 描述 |
|------|------|
| 身份 | A股短线投资者，日均持仓 5–15 只标的 |
| 使用频率 | 每个交易日盘后打开看板 1–2 次 |
| 熟练程度 | 熟悉看板现有页面，期望快速获取信息 |
| 核心诉求 | 第一时间知道当日盈亏全貌，快速回溯近期历史 |

### 2.2 用户故事列表

**US-1：未读提醒**
> 作为投资者，当我打开看板时，我想在右上角看到铃铛图标上的未读数字角标，以便知道有多少天的复盘摘要我还没查看。

**US-2：快速浏览列表**
> 作为投资者，我想点击铃铛后弹出消息列表，按日期从新到旧排列，每条显示标题和收益摘要预览，以便快速扫描近期盈亏情况。

**US-3：区分已读未读**
> 作为投资者，我想在消息列表中通过视觉样式（高亮/蓝点 vs 灰显/灰点）区分已读和未读消息，以便优先关注未读内容。

**US-4：查看详情**
> 作为投资者，当我点击某条消息时，我想进入该消息的详情视图，看到当日总收益、收益率、证券收益、基金收益、总市值、当日主题和主线板块，以便完成单日复盘。

**US-5：一键已读**
> 作为投资者，当我不想逐条查看时，我想点击"全部已读"按钮，一次性将所有未读消息标记为已读，以便快速消除数字角标。

**US-6：状态持久化**
> 作为投资者，当我刷新页面或下次打开看板时，我希望已读/未读状态保持上次操作的结果，以便不用重复查看。

```
+-------------+     点击铃铛      +------------------+
| 任意主页面   | ---------------> | 消息中心面板（覆盖层+面板）|

                                        |

                               +------------------+
                               |   消息列表视图    |

                    v---------------------
              +------------------+

              |  - 收益核心指标   |

              |  - 返回按钮       |
              +------------------+
```

### 4.2 信息架构

```
消息中心面板
├── 头部区域
│   ├── 标题"消息中心"
│   └── "全部已读"按钮（仅当存在未读时显示或可用）
├── 内容区域（视图切换）
│   ├── 消息列表视图（默认）
│   │   └── 消息条目 × N
│   │       ├── 状态指示点（蓝/灰）
│       ├── 标题
│       ├── 收益摘要预览
│       └── 日期
│   └── 消息详情视图
│       ├── 返回按钮 + 日期标题
│       ├── 收益总览卡片
│       │   ├── 总收益金额
│       │   ├── 总收益率
│       │   └── 总市值
│       ├── 分类收益
│       │   ├── 证券收益 + 收益率
│       │   └── 基金收益 + 收益率
│       └── 复盘标签
│           ├── 当日主题
│           └── 主线板块
└── （空状态：无消息数据时显示提示）
```

### 4.3 交互规则

- **面板层级**：消息中心面板为全局最高层级（z-index 高于页面内所有固定元素），但低于系统级弹窗（如数据加密密码框）。
- **面板打开时**：背景页面不随滚动条滚动，禁止 body 滚动；仅面板内部可滚动。
- **视图切换**：列表视图与详情视图在同一面板内通过动画切换（如左右滑动或淡入淡出），面板本身不关闭。
- **返回行为**：详情视图点击返回，回到列表视图并恢复之前的滚动位置。

## 5. 功能详细说明

### 5.1 F1：入口角标

| 维度 | 说明 |
|------|------|
| **输入** | 页面初始化时从数据源读取消息数组；从本地存储读取已读状态映射。 |
| **处理** | 计算消息数组中未被标记为已读的消息数量，得到未读数 N。若 N > 0，渲染红色圆形角标，角标内显示数字 N。 |
| **输出** | 铃铛图标右上角显示角标（N > 0）或不显示（N = 0）。 |
| **异常** | 本地存储读取失败（如存储被禁用）：按全部未读处理，角标显示全量消息数，并在控制台输出警告。 |

**业务规则**

- 角标数字最大显示 99，超过时显示"99+"。
- 铃铛图标常驻于页面右上角导航栏，不随页面滚动消失。
- 角标颜色为红色（#FF4D4F 或设计规范中的错误色），数字为白色。

---

### 5.2 F2：面板弹出

| 维度 | 说明 |
|------|------|
| **输入** | 用户点击铃铛图标。 |
| **处理** | 页面 body 添加 overflow: hidden 禁止背景滚动；渲染半透明黑色遮罩层（opacity 约 0.4）；从右侧滑入消息面板（transform: translateX 动画，时长 250ms）。 |
| **输出** | 面板完全展开，默认展示消息列表视图。 |
| **异常** | 若面板已打开时再次点击铃铛，不做任何操作（避免重复渲染）。 |

**交互细节**

- 遮罩层点击关闭面板（见 F9）。
- 面板宽度在桌面端固定为 400px，在移动端为屏幕宽度的 85%（最大 400px）。
- 动画采用 ease-out 缓动函数。

---

### 5.3 F3：消息列表

| 维度 | 说明 |
|------|------|
| **输入** | 消息数组（来源：portfolio.json 中的 messages 字段或内置 JS 变量）。 |
| **处理** | 按消息日期字段降序排序（新到旧）；遍历渲染为列表条目；每个条目绑定点击事件。 |
| **输出** | 渲染可滚动的消息列表。 |
| **异常** | 消息数组为空或不存在时，展示空状态（见 F10）；排序时若日期格式异常，将该条目标记为"日期异常"并排在末尾。 |

**列表条目结构**

| 字段 | 类型 | 说明 |
|------|------|------|
| 状态点 | 视觉元素 | 左侧 8px 圆点，未读蓝色，已读灰色 |
| 标题 | string | 消息标题，如"2026-06-30 复盘" |
| 摘要预览 | string | 收益摘要，如"总收益 +1,234 元 (+2.35%)" |
| 日期 | string | 格式 YYYY-MM-DD，位于条目右侧或底部 |

---

### 5.4 F4：已读状态视觉

| 维度 | 说明 |
|------|------|
| **输入** | 单条消息的已读标记（boolean）。 |
| **处理** | 若已读为 false：状态点为蓝色（#1890FF），条目背景使用轻微高亮色（如 #F0F5FF）；若已读为 true：状态点为灰色（#BFBFBF），背景为白色/透明。 |
| **输出** | 列表中每条消息的渲染样式反映其已读状态。 |
| **异常** | 已读标记字段缺失时，默认按未读处理。 |

---

### 5.5 F5：消息详情

| 维度 | 说明 |
|------|------|
| **输入** | 用户点击某条消息；传入该消息的完整数据对象。 |
| **处理** | 面板内视图从列表切换为详情；详情页按模块渲染收益数据；收益金额保留两位小数，收益率保留两位小数并带 % 符号。 |
| **输出** | 展示完整的消息详情页。 |
| **异常** | 若消息数据对象缺少某个字段（如基金收益缺失），该模块显示"--"或隐藏该模块，不报错。 |

**详情页字段清单**

| 字段 | 类型 | 格式要求 |
|------|------|----------|
| 日期标题 | string | YYYY-MM-DD 星期X |
| 总收益 | number | 千分位分隔，保留两位小数，前缀 +/- |
| 总收益率 | number | 保留两位小数，带 %，正值前加 + |
| 证券收益 | number | 同上 |
| 证券收益率 | number | 同上 |
| 基金收益 | number | 同上 |
| 基金收益率 | number | 同上 |
| 总市值 | number | 千分位分隔，保留两位小数 |
| 当日主题 | string | 纯文本，如"科技成长反弹" |
| 主线板块 | string | 纯文本，如"半导体、AI算力" |

**状态联动**

用户点击进入详情时，若该条消息为未读，自动将其标记为已读，同步更新本地存储和列表中的状态点。

---

### 5.6 F6：状态持久化

| 维度 | 说明 |
|------|------|
| **输入** | 用户将某条消息标记为已读（点击进入详情或点击全部已读）。 |
| **处理** | 将消息唯一标识（id 或 date）写入已读状态映射对象；将整个映射对象序列化后存入浏览器本地存储。 |
| **输出** | 本地存储更新成功。 |
| **异常** | 本地存储写入失败（如存储空间满、隐私模式）：在内存中保留已读状态，当前会话有效，页面刷新后丢失；控制台输出警告。 |

**存储结构**

```json
{
  "msg_read_v1": {
    "2026-06-30": true,
    "2026-06-29": true
  }
}
```

- 存储键名：`msg_read_v1`（带版本号，便于未来迁移）。
- 值为对象，键为消息唯一标识，值为 boolean true。
- 不存储 false 值，未出现的键视为未读。

---

### 5.7 F7：全部已读

| 维度 | 说明 |
|------|------|
| **输入** | 用户点击"全部已读"按钮。 |
| **处理** | 遍历当前消息数组，将所有未读消息的标识写入已读映射；更新本地存储；将角标数字归零；列表中所有条目同步变为已读视觉状态。 |
| **输出** | 角标消失，列表无高亮条目。 |
| **异常** | 若当前无未读消息，按钮可置为禁用状态或点击无响应。 |

---

### 5.8 F8：返回列表

| 维度 | 说明 |
|------|------|
| **输入** | 用户在详情视图点击返回按钮。 |
| **处理** | 视图从详情切换回列表；恢复列表滚动位置至进入详情前的位置。 |
| **输出** | 列表视图重新可见。 |
| **异常** | 无。 |

---

### 5.9 F9：遮罩关闭

| 维度 | 说明 |
|------|------|
| **输入** | 用户点击面板外遮罩层，或按下 Escape 键。 |
| **处理** | 面板向右侧滑出（动画 200ms）；移除遮罩层；恢复 body 滚动。 |
| **输出** | 面板关闭，用户回到原页面。 |
| **异常** | 无。 |

---

### 5.10 F10：空状态

| 维度 | 说明 |
|------|------|
| **输入** | 消息数组为空或数据源未返回消息字段。 |
| **处理** | 列表区域渲染空状态占位图和文案"暂无消息"。 |
| **输出** | 展示空状态，不展示列表。 |
| **异常** | 无。 |

## 6. 非功能需求

### 6.1 性能

| 指标 | 要求 |
|------|------|
| 面板首次打开时长 | 从点击铃铛到面板完全展开，≤ 300ms（在主流桌面设备上）。 |
| 列表渲染 | 100 条消息内无卡顿，滚动帧率 ≥ 50fps。 |
| 状态读取 | 本地存储读取在页面初始化时同步完成，不阻塞首屏渲染。 |
| 动画流畅度 | 面板滑入/滑出动画使用 transform 和 opacity，避免触发重排。 |

### 6.2 兼容性

| 维度 | 要求 |
|------|------|
| 浏览器 | Chrome、Firefox、Safari、Edge 最新 2 个主版本。 |
| 移动端 | iOS Safari、Android Chrome，支持触摸滑动和点击。 |
| 分辨率 | 适配 375px（手机）到 1920px（桌面）宽度，面板宽度按 6.2 节规则响应。 |
| 本地存储 | 若浏览器禁用 localStorage 或处于隐私模式，功能降级为仅当前会话记忆已读状态。 |

### 6.3 安全

| 维度 | 要求 |
|------|------|
| 数据敏感级 | 消息中心展示的收益数据为用户个人持仓信息，属于敏感数据。但本功能仅读取本地静态数据，不通过网络传输。 |
| XSS 防护 | 消息标题、摘要、主题、板块等文本字段在渲染前进行 HTML 转义，防止数据源被篡改时产生 XSS。 |
| 存储安全 | 已读状态仅保存在用户本地浏览器，不包含账户凭证或交易密码，无需额外加密。 |

## 7. 数据模型

### 7.1 消息对象（Message）

```typescript
interface Message {
  /** 消息唯一标识，建议格式为 YYYY-MM-DD */
  id: string;

  /** 消息日期 */
  date: string; // YYYY-MM-DD

  /** 消息标题，如"2026-06-30 复盘" */
  title: string;

  /** 收益摘要预览文本，用于列表展示 */
  summary: string; // 如"总收益 +1,234 元 (+2.35%)"

  /** 当日总收益金额（元） */
  totalProfit: number;

  /** 当日总收益率（小数，如 0.0235 表示 2.35%） */
  totalProfitRate: number;

  /** 证券收益金额（元） */
  stockProfit: number;

  /** 证券收益率（小数） */
  stockProfitRate: number;

  /** 基金收益金额（元） */
  fundProfit: number;

  /** 基金收益率（小数） */
  fundProfitRate: number;

  /** 当日总市值（元） */
  totalMarketValue: number;

  /** 当日主题文本 */
  theme: string;

  /** 当日主线板块文本 */
  mainSector: string;
}
```

### 7.2 已读状态存储（ReadStatus）

```typescript
interface ReadStatusMap {
  [messageId: string]: boolean;
}
```

**存储约定**

- 键名：`msg_read_v1`
- 存储位置：`localStorage`
- 序列化格式：JSON 字符串
- 只有值为 `true` 的键被视为已读；未出现的键视为未读。

### 7.3 数据源约定

消息数组从以下任一来源注入：

- `portfolio.json` 中的 `messages` 字段（推荐）。
- 页面内置 JS 变量 `window.__MESSAGES__`。

数据源在页面初始化时加载，消息中心仅读取，不修改源数据。

## 8. 风险与依赖

### 8.1 依赖项

| 编号 | 依赖内容 | 影响范围 | 状态 |
|------|----------|----------|------|
| D1 | portfolio.json 或内置变量需提供符合 7.1 节结构的 `messages` 数组 | 消息中心无数据则展示空状态 | 需确认 |
| D2 | 看板整体布局需为消息中心面板预留 z-index 和定位上下文 | 面板可能被其他固定元素遮挡 | 需确认 |
| D3 | 看板现有路由/状态管理需支持面板打开时不丢失页面状态 | 面板关闭后用户应回到原上下文 | 已满足（单页无路由） |

### 8.2 风险清单

| 编号 | 风险描述 | 概率 | 影响 | 应对策略 |
|------|----------|------|------|----------|
| R1 | 本地存储被用户手动清除或浏览器隐私模式限制写入 | 中 | 已读状态丢失，用户需重新查看 | 接受降级；状态仅存本地，不视为核心数据 |
| R2 | 消息数据量随时间增长导致列表过长 | 低 | 渲染性能下降，滚动卡顿 | 列表使用虚拟滚动或分页加载（P2 需求） |
| R3 | 用户同时打开多个浏览器标签页，各标签内存中的已读状态不同步 | 中 | 角标计数不一致 | 每次打开面板时从 localStorage 重新读取状态 |
| R4 | 消息日期格式不统一导致排序异常 | 低 | 列表顺序错乱 | 在数据解析层统一校验日期格式，异常日期排末尾 |

## 9. 验收标准

### 9.1 功能验收

| 编号 | 验收项 | 通过标准 |
|------|--------|----------|
| AC-1 | 入口角标 | 未读消息 > 0 时铃铛显示红色数字角标；= 0 时不显示；数字与未读数一致。 |
| AC-2 | 面板打开 | 点击铃铛后面板从右侧滑入，背景出现遮罩且页面不可滚动；动画流畅无抖动。 |
| AC-3 | 列表排序 | 消息按日期从新到旧排列；日期格式异常的消息排在最下方并标注。 |
| AC-4 | 已读视觉 | 未读消息左侧为蓝点且背景高亮；已读为灰点且背景正常；视觉区分明显。 |
| AC-5 | 详情展示 | 点击消息后进入详情，展示总收益、收益率、证券收益、基金收益、总市值、主题、板块；数值格式化正确。 |
| AC-6 | 自动已读 | 从列表点击进入详情后，该条消息自动标记为已读；返回列表后蓝点变灰点，角标数字减 1。 |
| AC-7 | 全部已读 | 点击"全部已读"后，所有消息变为已读状态，角标消失，localStorage 中对应全部写入 true。 |
| AC-8 | 状态持久化 | 标记已读后刷新页面，已读消息仍显示为已读；未读消息仍显示为未读。 |
| AC-9 | 面板关闭 | 点击遮罩或按 Escape 键，面板关闭，页面恢复可滚动。 |
| AC-10 | 空状态 | 数据源无消息时，列表区域展示"暂无消息"提示，不报错。 |
