## 1. 设计原则

消息中心的 UI 设计遵循以下核心原则，与主应用整体风格保持高度一致：

### 1.1 沉浸优先

消息中心以遮罩层 + 面板的浮层形式呈现，不打断用户当前浏览上下文。面板从视觉层级上高于主内容区，但操作完成后可一键关闭，保持用户的「位置感」。

### 1.2 信息密度适中

列表页采用「标题 + 预览 + 时间」的三行结构，单条消息高度约 72px，确保一屏可展示 6-8 条消息，减少滚动负担。详情页采用「键值对」纵向排列，数据项之间留有充足呼吸感。

### 1.3 状态显性化

未读与已读状态的差异必须一眼可辨：
- **未读消息**：左侧有醒目圆点标记，背景色略深，标题文字使用主色高亮
- **已读消息**：无圆点，背景色与面板底色融合，标题使用次要文字色

### 1.4 即时反馈

- 点击消息后角标数字即时减少
- 点击「全部已读」后列表项状态同步变化，无需二次刷新
- Hover 态提供轻微背景变化，确认可点击区域

## 3. 字体与字号规范

消息中心复用主应用字体栈与字号体系。

### 3.1 字体栈

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
```

### 3.2 字号层级

| 层级 | 字号 | 字重 | 行高 | 使用位置 |
|------|------|------|------|---------|
| 面板标题 | `0.95rem` | 700 | 1.4 | 面板头部「消息中心」 |
| 消息标题（未读） | `0.88rem` | 600 | 1.35 | 列表项标题 |
| 消息标题（已读） | `0.88rem` | 400 | 1.35 | 列表项标题 |
| 消息预览 | `0.82rem` | 400 | 1.4 | 列表项预览文本 |
| 消息时间 | `0.78rem` | 400 | 1.3 | 列表项右侧日期 |
| 统计栏文字 | `0.78rem` | 400 | 1.3 | 顶部「共 N 条通知」/「全部已读」 |
| 详情标题 | `1.05rem` | 700 | 1.4 | 详情页消息标题 |
| 详情键名 | `0.85rem` | 400 | 1.5 | 详情页左侧标签 |
| 详情键值 | `0.95rem` | 700 | 1.5 | 详情页右侧数值 |
| 空状态提示 | `0.88rem` | 400 | 1.6 | 「暂无消息」 |
| 角标数字 | `0.7rem` | 700 | 1 | 铃铛角标 |

### 3.3 数字与金额显示规范

- 所有金额统一保留 2 位小数，使用 `toFixed(2)` 格式化
- 正数前显式添加 `+` 号，负数保持 `-` 号
- 金额单位统一为「元」，百分比统一为 `%`，紧跟数值后不加空格
- 大数值使用千分位分隔（`toLocaleString()`）

## 4. 组件规范

### 4.1 铃铛入口

**位置**：页面右上角，`position: fixed`，`top: 14px`，`right: 56px`（在版本按钮左侧）  
**层级**：`z-index: 900`

```
┌─────────────────────┐
│        🔔           │  ← 铃铛图标，字号 1.1rem
│   ┌─────────────┐   │
│   │     3       │   │  ← 角标（未读数 > 0 时显示）
│   └─────────────┘   │
└─────────────────────┘
```

**样式细节**：
- 容器复用 `.version-button` 类：宽 30px，高 30px，圆形，`border-radius: 50%`
- 边框：`1px solid var(--yellow)`
- 背景：`var(--yellow-bg)`
- 文字色：`var(--yellow)`
- 布局：`inline-flex`，水平垂直居中
- 无文字阴影，无投影，保持扁平风格

### 4.2 角标（Badge）

**显示条件**：未读消息数量 > 0  
**位置**：铃铛按钮右上角，绝对定位，向右上方溢出

```css
.msg-badge {
  position: absolute;
  top: -6px;
  right: -6px;
  min-width: 18px;
  height: 18px;
  padding: 0 5px;
  border-radius: 9px;
  background: #ff1744;
  color: #ffffff;
  font-size: 0.7rem;
  font-weight: 700;
  text-align: center;
  line-height: 18px;
  display: none; /* 未读为 0 时隐藏 */
}
```

**行为**：
- 数字 > 99 时显示为 `99+`
- 数字变化时无动画，即时切换
- 从无到有时直接显示，不采用渐显动画（保持简洁）

## 5. 交互规范

### 5.1 点击交互

| 元素 | 点击行为 | 反馈 |
|------|---------|------|
| 铃铛按钮 | 打开消息中心面板 | 面板浮现，同时渲染列表 |
| 遮罩层背景 | 关闭面板 | 面板消失，角标保持当前状态 |
| 关闭按钮 | 关闭面板 | 同上 |
| 消息列表项 | 打开该消息详情 | 该项标记已读，角标减 1，内容切换为详情 |
| 「全部已读」按钮 | 所有消息标记已读 | 角标隐藏，列表项全部变为已读样式 |
| 返回按钮 | 从详情回到列表 | 内容区切换回列表，保持滚动位置（当前未做，可迭代） |

### 5.2 Hover 交互

| 元素 | Hover 效果 |
|------|-----------|
| 铃铛按钮 | 边框色加深，可添加 `transform: scale(1.05)`（当前未做，可选） |
| 消息列表项 | 背景色变为 `--card-hover`，过渡时间 `0.15s` |
| 关闭按钮 | 边框色变 `--yellow`，文字色变 `--yellow` |
| 「全部已读」按钮 | 文字可加下划线或亮度提升（当前为简单文字按钮） |
| 返回按钮 | 文字色从 `--text-dim` 变为 `--text` |

### 5.3 打开/关闭动画

**当前版本**：采用即时切换（`display: none` / `flex` + `opacity` 可选项），不引入复杂动画，保证低端设备流畅度。

**建议增强动画（可选迭代）**：

```css
/* 遮罩层淡入 */
.msg-overlay {
  opacity: 0;
  transition: opacity 0.2s ease;
}
.msg-overlay.active {
  opacity: 1;
}

/* 面板从下方滑入 + 淡入 */
.msg-panel {
  transform: translateY(20px) scale(0.98);
  opacity: 0;
  transition: all 0.25s cubic-bezier(0.16, 1, 0.3, 1);
}
.msg-overlay.active .msg-panel {
  transform: translateY(0) scale(1);
  opacity: 1;
}
```

动画原则：
- 持续时间不超过 300ms
- 使用 `transform` 与 `opacity` 触发 GPU 加速
- 支持 `prefers-reduced-motion` 媒体查询， motion 敏感用户可关闭动画

## 7. 设计稿说明（文字描述各状态）

### 7.1 状态一：无消息（空状态）

**触发条件**：`DAYS` 数组为空或尚无交易日数据。

**视觉描述**：
- 面板头部正常显示「消息中心 + 关闭按钮」
- 内容区中央显示文字「暂无消息」
- 文字颜色 `var(--text-dim)`，字号 `0.88rem`
- 不显示统计栏
- 铃铛角标隐藏

### 7.2 状态二：有未读消息（列表页）

**触发条件**：打开消息中心，存在未读消息。

**视觉描述**：
- 面板头部下方显示统计栏：左侧「共 N 条通知」（`--text-dim`），右侧「全部已读」按钮（`--blue`）
- 列表第一项（最新）左侧有黄色圆点，背景色略深于面板
- 标题字重 600，预览文字正常显示
- 右侧显示日期 `MM-DD`
- 铃铛按钮右上角显示红色角标，数字为未读数
- 列表项之间以极淡分割线分隔
