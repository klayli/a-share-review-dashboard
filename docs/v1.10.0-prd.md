# A股短线复盘看板 v1.10.0 正式版 - 产品需求文档（PRD）

> 版本：v1.0  
> 编写日期：2026-07-01  
> 状态：待评审

---

## 1. 需求背景与目标

### 1.1 背景
当前版本为 `v1.10.0-preview2`，存在以下三个核心问题待解决：
1. 版本发布未同步更新版本说明（`versions.json` 仍停留在 v1.9.8）
2. 新功能未经预览确认即直接部署，违反用户要求的"先预览、后发布"流程
3. 今天是 2026-07-01（新交易日），但页面仍展示 2026-06-30 的数据，未显示今天的实时收益

### 1.2 目标
- 建立规范的版本说明管理机制
- 建立"预览 → 确认 → 发布"的标准化部署流程
- 实现新交易日的自动识别与实时收益计算
- 建立交易时段定时刷新机制（9:30-15:15，每10秒）

## 3. 需求详细说明

### 3.1 R1 - 版本说明管理
**描述**：`versions.json` 需包含当前版本 `v1.10.0` 的完整信息。

**验收标准**：
- `current` 字段更新为 `v1.10.0`
- 新增 `v1.10.0` 版本条目，包含：version、date、summary、cache、deploy_url
- summary 清晰描述本期改动：页面路由系统、数据加密保护、Tab联动过滤、实时收益刷新
- 版本弹窗中正确显示 `v1.10.0（当前）`

### 3.2 R2 - 部署流程管控
**描述**：建立强制性的预览-确认-发布流程。

**流程定义**：
1. 开发完成 → 本地启动 HTTP 服务验证
2. 使用浏览器工具全面测试（首页、持仓页、市场页、Tab切换、图表显示）
3. 向用户提供本地预览 URL
4. 用户确认无问题后，执行 wrangler deploy
5. 部署成功后更新 `versions.json` 中的 `deploy_url`

**验收标准**：
- 本期所有修改必须在用户明确回复"确认发布"后才执行 wrangler deploy
- 部署前必须完成全量功能验证

### 3.3 R3 - 新交易日数据初始化
**描述**：当 `todayText()` 不在 `DAYS` 数组中时，自动初始化今日数据。

**初始化逻辑**：
1. 复制 `latestDay()` 的数据作为模板
2. 更新 `date` 为今天，`weekday` 为今日星期
3. `price` 初始化为 `latestDay` 的收盘价（即昨收）
4. `prev` 设为 `latestDay` 的 `price`（用于计算涨跌）
5. `theme`、`sectorName`、`summary` 等文案字段设为空字符串
6. `limitUp`、`volume` 等数值字段设为 0 或空
7. 将新数据 push 到 `DAYS` 末尾
8. `currentDate` 自动切换到今天

**验收标准**：
- 2026-07-01 首次加载时，`DAYS` 自动包含今天数据
- `currentDate` 默认选中今天
- 页面正常渲染，不报错

### 3.4 R4 - 实时收益计算
**描述**：通过新浪行情 API 获取实时价格，计算今日实时盈亏。

**计算逻辑**：
1. 获取持仓股票代码列表
2. 调用 `loadSinaQuotes` 获取实时行情
3. 今日市值 = Σ(持仓股数 × 实时价格)
4. 昨日市值 = Σ(持仓股数 × 昨收价格)
5. 当日盈亏 = 今日市值 - 昨日市值
6. 更新今日数据对象的 `price` 字段为实时价格
7. 触发页面重渲染

**基金处理**：
- 基金无实时 API，当日盈亏使用 `FUND_PORTFOLIO` 中的 `dailyIncome`
- 如果 `dailyIncome` 未设置，显示为 "—"

**验收标准**：
- 交易时段内，"当日总收益"显示实时计算值
- 价格随行情 API 返回值实时更新
- 非交易时段显示 "非交易刷新时段，数据固定"

### 3.5 R5 - 分时盈亏曲线动态构建
**描述**：交易时段内逐步构建 `INTRADAY_PNL` 数组。

**构建逻辑**：
1. 如果 `INTRADAY_PNL` 为空（新交易日），初始化一个空数组
2. 每次 `refreshLivePnL` 执行时：
   - 计算当前时间点的总盈亏（证券+基金）
   - 如果当前分钟已存在记录，更新该记录
   - 否则追加新记录 `{time: 'HH:MM', value: 盈亏金额}`
3. `renderCharts` 自动使用最新的 `INTRADAY_PNL` 渲染曲线

**验收标准**：
- 9:30 首次刷新后，intraday 曲线显示第一个点
- 随着刷新进行，曲线逐步延伸
- 15:15 后曲线定格，显示全天走势

### 3.6 R6 - 定时任务调整
**描述**：调整交易时段判断和刷新频率。

**调整内容**：
- `isTradingRefreshWindow`：开始时间从 9:15 改为 **9:30**
- `startLiveRefresh`：间隔从 5000ms（5秒）改为 **10000ms**（10秒）
- 结束时间保持 15:15 不变

**验收标准**：
- 9:30 前不触发刷新
- 9:30-15:15 每10秒刷新一次
- 15:15 后停止刷新

### 3.7 R7 - 当日盈亏归因适配
**描述**：新交易日缺少板块/主题数据时，归因图展示友好提示。

**验收标准**：
- 当 `theme` 或 `sectorName` 为空时，归因图显示 "今日暂无板块归因数据"
- 不影响其他图表正常显示

## 4. 非功能性需求

| 类型 | 要求 |
|------|------|
| 性能 | 定时刷新不阻塞页面交互；API 调用失败时优雅降级 |
| 兼容性 | 保持纯前端实现，不引入后端依赖 |
| 可靠性 | 新浪 API 失败时保留上一次成功数据，不显示错误弹窗 |
| 可维护性 | 版本说明与代码版本号统一管理，避免遗漏 |

---

## 5. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 新浪 API 跨域受限 | 实时行情获取失败 | 已有 JSONP 方案，失败时静默降级 |
| 非交易日打开页面 |  today 数据被错误创建 | 仅在 isTradingRefreshWindow 返回 true 时创建 today 数据 |
| 基金无实时净值 | 当日盈亏不完整 | 明确标注基金部分为 T-1 数据 |
